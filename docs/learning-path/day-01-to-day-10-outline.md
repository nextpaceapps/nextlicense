# 10-Day Student Track (NextLicense)

1. Environment setup, run API + Dashboard, open first PR with setup notes.
2. API: strengthen input validation for license creation endpoint.
3. Dashboard: add client-side and server-side validation for product creation flow.
4. API: implement license status transition guardrails (active/expired/revoked).
5. Dashboard: improve error handling UX for API failures.
6. API tests: add unit tests for validation and status transitions.
7. Frontend tests: add component tests for Products or Licenses screens.
8. Integration: end-to-end flow check for product -> license -> validation.
9. Refactor: reduce duplicated validation logic and document conventions.
10. Production-style PR: implement a scoped feature with tests + rollout notes.

Each day must include: context, learning goal, SDLC stage, acceptance criteria, and reflection.
